## 요구사항 정의 (FR)

### FR-001: 쿼터별 기록 구분
- **우선순위**: Must
- **설명**: 모든 스탯(득점, 리바운드, 어시스트 등)이 쿼터별로 구분 기록되어 박스스코어에서 쿼터 필터링이 가능해야 함
- **인수 기준**:
  - Given 1쿼터에 A선수가 2점슛 5개 성공
  - When 박스스코어에서 "1Q" 탭 선택
  - Then A선수의 PTS = 10, FGM = 5 표시
  - And "전체" 탭에서는 전 쿼터 합산 표시
- **현재 상태**: `LocalPlayByPlays.quarter` 필드 존재하나, `LocalPlayerStats`는 쿼터별 분리 없음 (전체 합산만)
- **변경 포인트**:
  - (A) `LocalPlayerStats`에 `quarter` 컬럼 추가 → 쿼터별 row 생성 **OR**
  - (B) `LocalPlayByPlays`의 quarter 필드를 활용해 쿼터별 집계 쿼리 작성 (스키마 변경 최소화) ← **권장**
  - mybdr `MatchPlayerStat`에도 `quarterStats` JSON 필드 추가 or 쿼터별 집계 API
- **관련 화면**: 박스스코어, 쿼터종료 다이얼로그
- **비기능**: 쿼터 전환 시 집계 쿼리 100ms 이내

### FR-002: 박스스코어 슈팅 성공률 추가
- **우선순위**: Must
- **설명**: 박스스코어에 FG%, 3P%, FT% 컬럼 추가
- **인수 기준**:
  - Given A선수 FGM=5, FGA=10
  - When 박스스코어 확인
  - Then FG% = 50.0% 표시
  - And FGA=0이면 "-" 표시 (0 나누기 방지)
- **계산식** (FIBA 표준):
  ```
  FG% = FGM / FGA × 100
  3P% = 3PM / 3PA × 100
  FT% = FTM / FTA × 100
  ```
- **현재 상태**: mybdr `MatchPlayerStat`에 `field_goal_percentage`, `three_point_percentage`, `free_throw_percentage` 이미 존재. bdr_stat 박스스코어 UI에서 미표시
- **관련 화면**: 박스스코어
- **비기능**: 소수점 1자리 반올림 표시

### FR-003: 박스스코어 코트마진(+/-) 추가
- **우선순위**: Must
- **설명**: 각 선수가 코트에 있는 동안의 득실점차(+/-)를 박스스코어에 표시
- **인수 기준**:
  - Given A선수가 코트에 있는 동안 우리팀 +15점, 상대팀 +10점
  - When 박스스코어 확인
  - Then A선수의 +/- = +5 표시
- **계산식** (Coach 확인):
  ```
  +/- = (선수 ON-COURT 동안 팀 득점) - (선수 ON-COURT 동안 상대 득점)
  ```
  - 교체(substitution) 이벤트 + 득점 이벤트 기반
  - 선수 입장 시 score snapshot 저장
  - 득점마다 현재 ON-COURT 5인에게 +/- 누적
- **현재 상태**: `LocalPlayerStats.plusMinus` 필드 존재하나 **미연동** (항상 0). mybdr에도 `plus_minus` 필드 존재
- **관련 화면**: 박스스코어
- **비기능**: 실시간 업데이트, 교체 시 자동 계산

### FR-004: 샷클락 리셋 후 정지 동작
- **우선순위**: Must
- **설명**: 샷클락을 24초로 리셋한 후, 게임클락은 계속 흐르되 샷클락은 멈춘 상태로 대기. 기록원이 수동으로 시작 가능
- **인수 기준**:
  - Given 게임 진행 중 (양쪽 클락 작동)
  - When [24s] 리셋 버튼 터치
  - Then 샷클락 = 24초 표시 + 정지 상태
  - And 게임클락은 계속 흐름
  - And 기록원이 샷클락 시작 버튼으로 재시작
- **Coach 참고**: FIBA 규칙상 샷클락과 게임클락은 동시 작동이 원칙이나, **기록원 앱 운용상** 리셋 직후 잠시 정지는 실무적 요구 (조작 편의)
- **관련 화면**: 스코어보드 헤더

### FR-005: 샷클락/게임클락 +/- 조정 버튼
- **우선순위**: Must
- **설명**: 샷클락과 게임클락 표시 위아래에 +/- 물리 버튼 추가하여 시간 수동 조정 가능
- **인수 기준**:
  - Given 샷클락 = 18초
  - When [+] 버튼 탭
  - Then 샷클락 = 19초
  - When [-] 버튼 탭
  - Then 샷클락 = 17초
  - And 게임클락도 동일 방식 (+/- 1초 단위)
- **추가 조건**: 클락 정지 상태에서만 조정 가능 (실행 중 오조작 방지)
- **관련 화면**: 스코어보드 헤더

### FR-006: 샷클락 1/10초 표시 (5초 미만)
- **우선순위**: Must
- **설명**: 샷클락이 5초 미만일 때 1/10초(소수점 1자리)까지 표시
- **인수 기준**:
  - Given 샷클락 = 5.0초
  - When 카운트다운 진행
  - Then 표시: "5" → "4.9" → "4.8" → ... → "0.1" → "0.0"
  - And 5초 이상에서는 정수 표시: "24", "23", ...
- **Coach 확인**: FIBA 규칙 Art. 29 — **5초 미만**에서 1/10초 표시 (사장 요청은 10초 미만 1/100초였으나, FIBA 규격은 5초 미만 1/10초)
- **🛑 사장 확인**: FIBA 규격(5초 미만, 1/10초)으로 갈지, 사장 요청(10초 미만, 1/100초)으로 갈지?
  - **옵션 A**: FIBA 규격 — 5초 미만 1/10초 ← Coach 권장
  - **옵션 B**: 사장 요청 — 10초 미만 1/100초
  - **옵션 C**: 대회 설정으로 선택 가능하게 (game_rules에 옵션 추가)
- **관련 화면**: 스코어보드 헤더

### FR-007: 게임클락 1/10초 표시 (1분 이내)
- **우선순위**: Must
- **설명**: 게임클락이 1분(60초) 이내일 때 1/10초까지 카운트 및 조절 가능
- **인수 기준**:
  - Given 게임클락 = 59.9초
  - When 카운트다운 진행
  - Then 표시: "0:59.9" → "0:59.8" → ... → "0:00.0"
  - And 1분 이상에서는 기존 MM:SS 포맷
  - And +/- 조정 시 0.1초 단위로 조정
- **현재 상태**: `Timer.periodic(Duration(seconds: 1))` → `Duration(milliseconds: 100)`으로 변경 필요
- **관련 화면**: 스코어보드 헤더
- **비기능**: 100ms 타이머 → CPU 부하 모니터링 필요

### FR-008: 작전타임 버튼 배치
- **우선순위**: Must
- **설명**: 양팀 벤치 영역 근처에 작전타임 호출 버튼 배치. 각 팀 색상으로 구분
- **인수 기준**:
  - Given 경기 진행 중
  - When 홈팀 벤치 옆 [타임아웃] 버튼 탭
  - Then 홈팀 타임아웃 1회 차감
  - And 타임아웃 play-by-play 기록 생성
  - And 잔여 타임아웃 표시 업데이트
  - And 남은 타임아웃 0이면 버튼 비활성화
- **관련 화면**: 게임 레코딩 화면 (벤치 영역)

### FR-009: 팀파울 버튼 배치
- **우선순위**: Should
- **설명**: 작전타임 버튼 옆에 팀파울 현황 표시 및 수동 조정 버튼
- **인수 기준**:
  - Given 현재 쿼터 팀파울 = 3
  - When 팀파울 표시 확인
  - Then "3/5" 형태로 표시 (현재/보너스기준)
  - And 5 이상이면 "BONUS" 뱃지 표시
- **현재 상태**: `TeamFoulBonusWidget` 이미 존재 → 배치 위치만 변경
- **관련 화면**: 게임 레코딩 화면 (벤치 영역)

### FR-010: 대회 등록 시 작전타임 횟수 설정
- **우선순위**: Must
- **설명**: 대회 생성/수정 시 `game_rules` JSON에 작전타임 관련 설정 포함. bdr_stat에서 이 값을 읽어 적용
- **인수 기준**:
  - Given 대회 game_rules에 `timeouts_first_half: 2, timeouts_second_half: 3, timeouts_overtime: 1`
  - When bdr_stat에서 해당 대회 경기 시작
  - Then 전반 타임아웃 각 2회, 후반 3회, 연장 1회 적용
- **FIBA 기본값** (Coach 확인):
  ```json
  {
    "timeouts_first_half": 2,
    "timeouts_second_half": 3,
    "timeouts_overtime": 1,
    "timeout_duration_seconds": 60,
    "max_timeouts_last_2min": 2
  }
  ```
- **현재 상태**:
  - mybdr: `Tournament.game_rules` (JSON) 필드 존재 — 내부 구조 미정의
  - bdr_stat: `GameRulesDefaults.totalTimeouts = 4` 하드코딩 + `fromJson()` 메서드 존재
  - mybdr 대회 생성 UI에 타임아웃 설정 필드 **미구현**
- **변경 포인트**:
  - mybdr: 대회 생성/편집 페이지에 game_rules 편집 UI 추가
  - mybdr: game_rules JSON 스키마 정의 및 API 응답에 포함
  - bdr_stat: `GameRulesDefaults.fromJson()` 확장하여 새 필드 파싱
  - bdr_stat: `RecordingState` 초기화 시 대회 game_rules에서 타임아웃 값 로드
- **관련 화면**: mybdr 대회 관리 페이지, bdr_stat 경기 시작 화면

### FR-011: 로그 영역 축소
- **우선순위**: Must
- **설명**: 게임 로그 표시 영역을 현재의 1/2로 줄이고, 최근 3~4줄만 표시
- **인수 기준**:
  - Given 10개의 이벤트 기록
  - When 로그 영역 확인
  - Then 최근 3~4개 이벤트만 표시
  - And 로그 영역 높이가 기존의 50%
  - And 나머지 공간은 코트/벤치 영역에 할당
- **현재 상태**: `LiveGameLog` / `EventLog` 위젯 — 전체 높이 고정 없이 스크롤
- **관련 화면**: 게임 레코딩 화면

### FR-012: 에어커맨드 UX 변경 (터치 기반 원형 메뉴)
- **우선순위**: Must
- **설명**: 드래그앤드롭 방식을 터치 기반 원형 메뉴로 변경. 선수 터치 시 아래 순서로 에어커맨드 표시
- **메뉴 레이아웃**:
  ```
            [파울]          ← 최상단 (12시 방향)
              |
              |→ 서브메뉴: 오펜스파울 / 디펜스파울 / 테크니컬 / U파울 / 플래그런트1 / 플래그런트2

  시계 방향 (3시→):
    OR → DR → AST → BLK → STL → TO
  ```
- **인수 기준**:
  - Given 경기 진행 중, 코트에 선수 표시
  - When A선수 터치
  - Then 원형 에어커맨드 메뉴 표시 (200ms 이내)
  - When [파울] 버튼 터치
  - Then 파울 종류 서브메뉴 표시 (오펜스/디펜스/테크니컬/U/플래그런트1/플래그런트2)
  - When [OR] 버튼 터치
  - Then 공격 리바운드 즉시 기록
- **현재 상태**: `RadialActionMenu` 존재하나 다른 레이아웃. 재배치 필요
- **파울 서브타입 매핑**:
  - 오펜스 파울 → `actionSubtype: 'offensive'`
  - 디펜스 파울 → `actionSubtype: 'personal'` (기존 개인파울)
  - 테크니컬 파울 → `actionSubtype: 'technical'`
  - U 파울 → `actionSubtype: 'unsportsmanlike'`
  - 플래그런트 1 → `actionSubtype: 'flagrant'` + `isFlagrant: true`
  - 플래그런트 2 → `actionSubtype: 'flagrant2'` (신규)
- **관련 화면**: 게임 레코딩 화면 (코트 영역)

### FR-013: 안내문 자동 소멸 (2초)
- **우선순위**: Must
- **설명**: 언두, 동기화 등 안내 메시지가 2초 후 자동으로 사라짐
- **인수 기준**:
  - Given 이벤트 기록 완료
  - When 안내 스낵바 표시
  - Then 2초 후 자동 소멸
  - And 소멸 전 사용자가 "실행 취소" 가능
- **현재 상태**: `UndoSnackbar.show()` duration = 3초, `showSuccess()` = 2초. **문제는 사라지지 않는 버그**
- **관련 화면**: 전역 (ScaffoldMessenger)
