# BDR Stat 현장 운영 매뉴얼

> 대회 기록원용 | 최종 수정: 2026-03-15

---

## 1. 사전 준비 (대회 전날)

### 필수 체크리스트

- [ ] 태블릿 충전 100%
- [ ] bdr_stat 앱 최신 버전 확인
- [ ] WiFi 연결 상태에서 **대회 데이터 다운로드**
  - 앱 실행 → 대회 연결 → "전체 데이터 다운로드"
  - 팀 목록 + 선수 목록 + 대진표 표시 확인
- [ ] 테스트 기록 1회 실행 (정상 동작 확인)
- [ ] 보조 배터리 + 충전 케이블 준비

### 장비 배정

| 코트 수 | 필요 태블릿 | 비고 |
|---------|-----------|------|
| 1코트 | 1대 | 기본 |
| 2코트 동시 진행 | 2대 | 코트당 1대 필수 |
| 3코트 이상 | 코트 수만큼 | 각 기기에 같은 대회 다운로드 |

> **중요**: 1대의 태블릿으로 2경기를 동시에 기록할 수 없습니다.

## 2. 당일 운영

### 경기 시작 전

1. 태블릿 전원 ON + 앱 실행
2. 경기 목록에서 해당 경기 선택
3. 선발 선수 지정
4. 충전 케이블 연결 (가능하면)

### 경기 중 기록

- 정상적으로 스탯 입력 (득점, 리바운드, 파울 등)
- **30초마다 자동 저장** — 별도 저장 버튼 누를 필요 없음
- 화면 상단 배너 확인:
  - 배너 없음 → 온라인, 실시간 전송 중
  - 🟡 "오프라인" → 로컬 저장 중 (정상)
  - 🔴 "배터리 부족" → 충전기 연결

### 경기 종료 후

1. 경기 종료 처리
2. MVP 선정 (선택)
3. 최종 스탯 확인
4. WiFi 연결 시 자동 업로드 (30초 이내 시작)
5. **"동기화 완료" 확인** 후 다음 경기 진행

## 3. 오프라인 운영 가이드

### 네트워크 없이 기록하기

**핵심: 네트워크가 없어도 기록은 정상 작동합니다.**

- 모든 데이터는 태블릿 내부에 저장됨
- 하루 8경기까지 문제없이 저장 가능 (용량 약 2MB)
- 네트워크 복구 시 자동으로 서버에 전송

### 관중/주최자 안내 문구 (복사해서 사용)

> 현장 네트워크 사정으로 실시간 스코어 업데이트가 지연되고 있습니다.
> 경기 종료 후 일괄 업데이트됩니다. 양해 부탁드립니다.

### 오프라인 시 주의사항

| 해도 됨 | 하면 안 됨 |
|---------|-----------|
| 경기 기록 계속하기 | ❌ 앱 삭제 (데이터 사라짐) |
| 앱 종료 후 재시작 | ❌ 태블릿 초기화 |
| 다른 경기로 전환 | ❌ 같은 경기를 다른 기기에서 동시 기록 |
| 배터리 충전 | |

## 4. 문제 상황 대응

### 앱이 갑자기 꺼졌어요

1. 앱을 다시 실행하세요
2. **"기록 중인 경기가 있습니다. 이어서 기록하시겠습니까?"** 팝업이 뜹니다
3. **[이어서 기록]** 버튼을 누르세요
4. 마지막 저장 시점(최대 30초 전)부터 이어서 기록됩니다

> 데이터는 유실되지 않습니다.

### 배터리가 방전됐어요

1. 충전 후 전원을 켜세요
2. 위의 "앱이 갑자기 꺼졌어요"와 동일하게 복구됩니다
3. 데이터는 안전하게 보존되어 있습니다

### WiFi가 안 돼요

1. **기록은 계속하세요** — 오프라인에서도 정상 동작합니다
2. 경기 종료 후 WiFi가 되는 곳에서 앱을 열어두면 자동 전송됩니다
3. 휴대폰 핫스팟도 사용 가능합니다

### 기록원을 교체해야 해요

1. 현재 기록원: 진행 중인 경기를 **종료하지 마세요**
2. 새 기록원에게 **같은 태블릿**을 넘기세요
3. 다른 태블릿을 사용해야 하면:
   - 이전 태블릿에서 WiFi 연결 → 데이터 업로드 확인
   - 새 태블릿에서 대회 데이터 다운로드 후 경기 시작

### 업로드가 실패했다고 나와요

1. WiFi 연결 상태를 확인하세요
2. 앱을 종료 후 재시작하세요
3. 자동으로 재시도됩니다 (최대 50회)
4. 계속 실패하면 운영자에게 연락하세요

### 같은 경기가 두 번 올라간 것 같아요

- 시스템이 자동으로 중복을 감지합니다
- 충돌 시 "로컬 데이터 사용 / 서버 데이터 사용" 선택 화면이 나타납니다
- 보통 **[로컬 데이터 사용]**을 선택하세요 (최신 기록)

## 5. 하루 일정 예시 (8경기)

```
08:00  태블릿 충전 확인 + 앱 실행 + WiFi 연결 확인
08:30  1경기 시작 → 기록
09:30  1경기 종료 → 동기화 확인
09:45  2경기 시작 → 기록
       ...
       (WiFi 장애 발생 — 오프라인 모드 자동 전환)
       ...
12:00  점심시간 — 태블릿 충전
       (WiFi 복구 시 자동 업로드)
13:00  5경기 시작
       ...
17:00  8경기 종료
17:10  WiFi 연결 → 미전송 데이터 일괄 업로드
17:20  "동기화 완료" 확인 → 업무 종료
```

## 7. 기술 참고 (운영자용)

- 로컬 저장: SQLite (Drift ORM) — 앱 삭제 시에만 데이터 소실
- 자동 저장 간격: 30초
- 동기화 재시도: 최대 50회 (지수 백오프, 최대 5분 간격)
- 배터리 경고: 20% 이하 (주의), 10% 이하 (위험)
- 1경기 데이터 크기: 약 100KB
- 8경기 누적: 약 2MB (태블릿 용량의 0.003%)
