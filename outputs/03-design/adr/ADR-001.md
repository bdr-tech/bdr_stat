# ADR-001: game_rules JSON 섹션별 구조화

| 항목 | 내용 |
|------|------|
| 상태 | 승인 |
| 작성일 | 2026-03-16 |
| 작성자 | Marcus (설계 에이전트) |
| 요구사항 | FR-006, FR-010 |

## 컨텍스트

`Tournament.game_rules` JSONB 컬럼에 Sprint 2 신규 키를 추가해야 한다.
- FR-006: `shot_clock_decimal_threshold_seconds`, `shot_clock_decimal_precision`
- FR-010: `timeouts_first_half`, `timeouts_second_half`, `timeouts_overtime`, `bonus_timeout_*`

기존 `GameRulesDefaults`는 단일 평탄 구조를 사용한다. 신규 키를 동일 구조로 추가하면 30개 이상의 키가 무분별하게 나열된다.

## 결정

`timing` / `shot_clock` / `fouls` / `timeouts` / `scoring` 5개 섹션으로 중첩 JSON 구조를 도입한다.

기존 `GameRulesDefaults`는 유지 (하위 호환). 신규 `GameRulesModel` 도입.

## 대안

A. 기존 평탄 구조 확장: 최소 변경이나 키 증가 시 관리 어려움.
B. 섹션별 중첩 구조 (채택): 관련 설정 그룹화, 안전한 확장.

## 결과

- `lib/domain/models/game_rules_model.dart` 신규 생성
- `GameRulesModel.fromJson()`: 섹션 없으면 FIBA 기본값 fallback
- 기존 `GameRulesDefaults.fromJson()` 유지 (deprecated 처리 않음)

## 트레이드오프

포기: 두 가지 파싱 경로 공존 기간 동안의 혼란 가능성.
얻음: 섹션 추가가 다른 섹션에 영향을 주지 않는 안전한 확장성.
