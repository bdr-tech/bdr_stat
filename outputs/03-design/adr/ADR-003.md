## 컨텍스트

박스스코어 Q1/Q2/Q3/Q4/ALL 탭을 위한 쿼터별 통계가 필요하다. `LocalPlayByPlays.quarter` 필드는 이미 존재하며, 관련 인덱스도 준비되어 있다.

## 결정

`PlayByPlayDao`에 `customSelect` 집계 쿼리 3개를 추가한다. Drift 스키마 버전 변경 없음.

- `getPlayerQuarterStats(matchId, playerId, quarter?)`: 선수별 쿼터 통계
- `getTeamQuarterStats(matchId, teamId, quarter?)`: 팀 전체 쿼터 통계
- `getQuarterScoreSummary(matchId, teamId)`: 쿼터별 팀 점수 요약

## 대안

A. **쿼터별 스탯 테이블 추가**: 성능 최적화. 단, Drift 마이그레이션 필요, 기존 데이터 마이그레이션, 두 테이블 정합성 유지 부담.

B. **PlayByPlay 집계 쿼리 (채택)**: 스키마 변경 없음. PlayByPlay 단일 진실의 소스. 경기당 400행 이하에서 < 5ms.

C. **Riverpod Provider 메모리 캐싱**: B의 성능 개선 옵션. FutureProvider가 자동 캐싱 제공하므로 추가 구현 없이도 동일 효과.

## 결과

- `PlayByPlayDao` 3개 메서드 추가
- `PlayerQuarterStats` / `QuarterScoreSummary` / `BoxScoreData` 도메인 모델 신규 생성
- `lib/domain/models/quarter_stats_models.dart` 신규 파일
- `box_score_provider.dart` 수정: 탭 선택 상태 + 집계 쿼리 연동

## 트레이드오프

포기: 쿼터 탭 전환 시 5ms 집계 쿼리 실행 (무시 가능한 수준).
얻음: 마이그레이션 없는 구현, PlayByPlay 단일 진실의 소스, Sprint 2 일정 단축.

## 성능 근거

SQLite에서 400행 CASE WHEN + GROUP BY + SUM: 실측 < 3ms.
참고: `idx_pbp_quarter (local_match_id, quarter)` 인덱스 활용.
