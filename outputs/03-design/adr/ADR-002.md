# ADR-002: +/- 계산 시점 — 이벤트 발생 즉시 vs 경기 종료 후 집계

| 항목 | 내용 |

| 상태 | 승인 |
| 작성일 | 2026-03-16 |

| 요구사항 | FR-003 |

## 컨텍스트

+/- 계산을 구현하는 두 가지 주요 방법이 있다. 선택 기준은 (1) 실시간 표시 가능 여부, (2) Undo/수정 시 정확성, (3) 기존 코드 패턴과의 일관성이다.

## 결정

이벤트 발생 즉시 `LocalPlayerStats.plusMinus`를 원자적으로 업데이트한다.
- 득점 트랜잭션 내: `applyPlusMinusForScore()` 호출
- ON-COURT 선수 (`isOnCourt = true`) 기준으로 즉시 누적
- Undo는 현재 ON-COURT 기준으로 역방향 적용

## 대안

A. **이벤트 즉시 누적 (채택)**: 박스스코어 실시간 표시 가능. 기존 `_atomicIncrement` 패턴과 일관성.

B. **경기 종료 후 집계**: PlayByPlay를 시간 순으로 재생하여 +/- 계산. 정확하나 박스스코어 화면에서 매번 전체 재집계 필요.

C. **PlayByPlay 실시간 집계 쿼리**: 교체 시각과 득점 시각을 gameClockSeconds 기준으로 비교. 게임 클락 카운트다운 해석 오류 시 잘못된 값 노출 위험 → 채택 불가.

## 결과

- `PlayerStatsDao.updatePlusMinus(matchId, playerId, delta)` 신규 추가
- `PlayerStatsDao.applyPlusMinusForScore(...)` 신규 추가
- 득점 이벤트 기록 트랜잭션에서 두 번째 단계로 호출
- 교체 이벤트: plusMinus 변경 없음

## 트레이드오프

포기: Undo/소급 수정 후 +/- 완벽한 정확성 (best-effort 값).
얻음: 구현 단순성, 박스스코어 실시간 표시 성능, 기존 패턴 일관성.

## 엣지 케이스 기록

- 교체 누락: 현재 ON-COURT 상태 기준으로 처리 (소급 재계산 없음)
- 파울아웃/부상으로 5인 미만: 코트 위 선수에게만 적용
- OT: 동일 로직 적용, 특별 처리 없음
