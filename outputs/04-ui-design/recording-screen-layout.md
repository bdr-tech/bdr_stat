## 1. 현재 레이아웃 분석

현재 `GameRecordingScreen`은 가로 방향 태블릿 기준으로 아래 구조를 사용한다:

- **헤더 영역**: `GameHeader` — 팀명/점수/쿼터 (flex 9) + 액션 버튼 (flex 3)
- **사이드바**: `LiveGameLogPanel` — 게임클락 + 샷클락 + 쿼터제어 + **로그 (Expanded)**
- **메인 코트**: `CourtWithPlayers` + `TeamPlayersPanel`
- **하단 벤치**: `BenchSection` — 양팀 벤치 (현재: 선수 아이콘만, 타임아웃/파울 없음)

### 1.1 문제점

| 문제 | 위치 | FR |
|------|------|-----|
| 작전타임 버튼 없음 | BenchSection | FR-008 |
| 팀파울 위젯 위치 불명확 | 없음/헤더 근처 | FR-009 |
| 로그 영역 높이 제한 없음 | LiveGameLogPanel | FR-011 |

## 3. 벤치 섹션 (BenchSection) 상세 레이아웃 — FR-008, FR-009

### 3.1 구성 요소 배치

각 팀 패널은 아래 세 레이어로 구성:

```
┌──────────────────────────────────────────┐
│  BENCH HOME                              │  ← 팀명 라벨 (기존)
│                                          │
│  ┌─────────────┐   ┌──────────────────┐  │
│  │  팀파울 표시 │   │  타임아웃 상태   │  │  ← Layer 1: 상태 표시
│  │  ●●●○○ 3/5 │   │  ■■□□  2잔여    │  │
│  │             │   │                  │  │
│  └─────────────┘   └──────────────────┘  │
│                                          │
│       [T/O 호출] (팀 컬러 버튼)          │  ← Layer 2: 타임아웃 버튼
│                                          │
│  [#5]  [#11]  [#23]  [#30]  [#33]       │  ← Layer 3: 선수 아이콘
└──────────────────────────────────────────┘
```

### 3.2 팀파울 표시 위젯 명세 (FR-009)

기존 `TeamFoulBonusWidget` 컴팩트 모드 활용 + 숫자 표기 추가.

```
정상 상태 (3/5):
┌─────────────────┐
│ FOULS  ●●●○○   │
│         3/5     │
└─────────────────┘

보너스 상태 (5+):
┌─────────────────────┐
│ FOULS  ●●●●● BONUS │  ← amber 컬러 border
│         5/5         │
└─────────────────────┘
```

- 컨테이너 크기: 너비 자동, 높이 48px
- 배경: `AppTheme.surfaceColor`
- 보너스 시: border `AppTheme.warningColor` 2px, 배경 `warningColor.withValues(alpha: 0.1)`

### 3.3 타임아웃 상태 표시 명세 (FR-008)

기존 `TimeoutRemainingWidget` 컴팩트 모드 개선.

```
2잔여 (전반 2/2 기준 — game_rules 값 사용):
┌────────────────────┐
│ T/O  ■■□□□        │  ← 홈팀 컬러 filled, 회색 empty
│       2 remaining  │
└────────────────────┘

0잔여:
┌────────────────────┐
│ T/O  □□□□□        │  ← 모두 비어있음
│       0 remaining  │  ← errorColor 텍스트
└────────────────────┘
```

### 3.4 타임아웃 호출 버튼 명세 (FR-008 핵심)

```
활성 상태 (잔여 > 0):
┌──────────────────────────────────────┐
│  [ timer_off 아이콘 ]  T/O 호출      │  ← 팀 컬러 배경 (homeTeamColor)
└──────────────────────────────────────┘
높이: 44px (터치 타겟)
배경: teamColor.withValues(alpha: 0.15)
border: teamColor.withValues(alpha: 0.3)
텍스트: teamColor, FontWeight.bold, 14px

비활성 상태 (잔여 = 0):
┌──────────────────────────────────────┐
│  [ timer_off 아이콘 ]  T/O 없음      │  ← 회색 비활성
└──────────────────────────────────────┘
배경: Colors.grey[800]
텍스트: Colors.grey[500]
onTap: null (비활성화)
```

#### 버튼 탭 동작
1. 타임아웃 잔여 -1 차감
2. play-by-play 타임아웃 기록 (`UndoableActionType.timeout`)
3. Undo 스낵바 표시 (2초)
4. 잔여 = 0 → 버튼 자동 비활성화

### 3.5 전반/후반 타임아웃 분리 표시 (FR-010 연동)

쿼터 전환 시 상태 갱신 규칙:
- Q1, Q2: `gameRules.timeoutsFirstHalf` 기준
- Q3, Q4: `gameRules.timeoutsSecondHalf` 기준
- OT: `gameRules.timeoutsOvertime` 기준

```
Q1 상태 (전반 2/2 기준):
[ T/O 2 | ■■ ]

Q3 진입 시 리셋 후 표시 (후반 3/3 기준):
[ T/O 3 | ■■■ ]  ← 전반과 별도 카운트
```

## 4. 로그 영역 축소 — FR-011

### 4.1 현재 구조

`LiveGameLogPanel._buildLogSection()`:
- `Expanded` → 사이드바 남은 공간 전부 차지
- 아이템 수 제한 없음 (`ListView.builder`)

### 4.2 Sprint 2 목표 구조

```
LIVE LOG 헤더 (고정 32px)
┌────────────────────┐
│ ● LIVE LOG    [n]  │  ← 32px
├────────────────────┤
│ [로그 항목 1] ←최신│  ─┐
│ [로그 항목 2]       │   │  각 항목 44px → 4줄 = 176px
│ [로그 항목 3]       │   │
│ [로그 항목 4]       │  ─┘
└────────────────────┘
총 높이: 32 + 176 + 패딩 16 = 약 224px (고정)
```

### 4.3 구현 명세

```dart
// _buildLogSection에서 Expanded 제거, 고정 높이로 변경

Container(
  height: 224,   // 고정 높이 (기존 Expanded 대체)
  decoration: ...,
  child: Column(
    children: [
      // 헤더 (기존 유지)
      _buildLogHeader(),  // 32px

      // 로그 리스트 — 최근 4개만, 스크롤 없음
      ListView.builder(
        shrinkWrap: true,
        physics: const NeverScrollableScrollPhysics(),
        padding: const EdgeInsets.all(8),
        itemCount: min(undoStack.actions.length, 4),   // 최대 4개
        reverse: true,
        itemBuilder: (context, index) {
          // 최신 4개만 노출
          final actions = undoStack.actions;
          final startIndex = max(0, actions.length - 4);
          final action = actions[startIndex + index];
          ...
        },
      ),
    ],
  ),
)
```

**주의**: 기존 `Expanded`를 제거하므로 사이드바 Column에서 `LiveGameLogPanel`을 `Flexible` 또는 고정 height로 감싸야 함.

## 5. 사이드바 전체 높이 배분

사이드바 전체 높이 = 화면 높이 - 헤더 - 벤치섹션.

```
사이드바 Column (ScrollView 내 또는 고정):
┌────────────────────────────┐
│ GAME CLOCK 섹션            │  ~120px (클락 표시 + +/- 버튼)
│ SHOT CLOCK 섹션            │  ~100px (클락 표시 + +/- 버튼 + 리셋)

│ LIVE LOG (고정 224px)      │  224px
└────────────────────────────┘
```

## 7. 전체 화면 상태별 명세

### 7.1 정상 상태

위 섹션 2의 레이아웃 그대로.

### 7.2 로딩 상태

`_isLoading = true` 시:
- 벤치 섹션: 선수 아이콘 자리에 `CircularProgressIndicator`
- T/O 버튼: 비활성 (회색)
- 팀파울 표시: `- / -`

### 7.3 빈 상태 (경기 시작 전, 로그 0건)

```
LIVE LOG:
┌────────────────────────────┐
│ ● LIVE LOG         [0]    │
│                            │
│   🏀 RECORDING...          │  ← 기존 빈 상태 유지
│   선수를 탭하여 시작하세요  │
│                            │
└────────────────────────────┘
높이: 224px 고정 유지 (빈 상태도 동일 크기)
```

### 7.4 에러 상태

타임아웃 호출 실패(DB 오류):
- 차감 롤백 (Undo 스택 활용)
- 스낵바: "타임아웃 기록 실패. 다시 시도하세요" (에러 색상)
- 버튼 비활성화 해제

### 7.5 보너스 진입 시

팀파울 5개 이상:
- 팀파울 위젯 배경 `warningColor.withValues(alpha: 0.1)`
- `BONUS` 배지 표시
- 추가 파울 시: 자유투 시퀀스 자동 트리거 (기존 로직 유지)
