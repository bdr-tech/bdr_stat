> **컨설턴트**: Victor (08-consultant)

## Executive Summary

BDR Tournament Recorder v1.0은 **릴리스 준비 완료** 상태입니다 (402개 테스트 통과, 모든 버그 수정 완료). 오프라인 우선 아키텍처와 빠른 입력 속도라는 핵심 목표를 잘 달성했습니다.

이 보고서에서는 v2.0 개발을 위한 **신규 기능 아이디어**, **UX 개선점**, **로드맵**을 제안합니다.

## Part 1: 신규 기능 아이디어 (RICE 우선순위 평가)

### 1.1 고우선순위 기능 (RICE Score: 80+)

#### A. 실시간 스코어보드 공유 (Score: 92)

| RICE 항목 | 점수 | 근거 |
|-----------|------|------|
| Reach | 9 | 관중, 선수 가족, 리그 관계자 모두 대상 |
| Impact | 10 | 농구 경기의 핵심 니즈 - "지금 몇 대 몇?" |
| Confidence | 9 | 기존 동기화 인프라 활용 가능 |
| Effort | 3 | 웹 뷰어 개발 필요 (2-3주) |

**기능 상세**:
```
- 고유 URL 생성: bdr.app/live/MATCH_ID
- 실시간 점수, 쿼터, 팀 파울 표시
- 최근 플레이 로그 스트리밍
- QR 코드 생성 (체육관 스크린 공유용)
- 오프라인 시 "마지막 동기화: 3분 전" 표시
```

**기술 구현 방향**:
- Rails API에 WebSocket 엔드포인트 추가
- 간단한 React/Next.js 웹 뷰어
- 기존 `SyncManager`의 성공 콜백에 푸시 연동

| RICE 항목 | 점수 | 근거 |

| Reach | 8 | 감독, 코치, 데이터 분석가 대상 |
| Impact | 10 | 차별화된 가치 - "이 경기 어땠어요?" 한 문장 요약 |
| Confidence | 8 | PlayByPlay 데이터 풍부 |
| Effort | 4 | LLM API 연동 필요 (3-4주) |

**기능 상세**:
```
- 경기 종료 후 자동 생성:
  "홈팀이 3쿼터 12-0 런으로 역전, 김철수 22점 16리바운드 더블더블"

- 핵심 모멘트 추출:
  - 최대 득점 런 (scoring run)
  - 클러치 타임 성과 (4쿼터 2분 이내)
  - 개인 하이라이트 (더블더블, 트리플더블)

- 소셜 미디어 공유용 카드 이미지 생성
```

**기술 구현 방향**:
- 경기 종료 시 `PlayByPlay` 데이터를 프롬프트로 변환
- OpenAI GPT-4o-mini API 호출 (비용 효율)
- 결과 캐싱하여 중복 호출 방지

## Part 2: 사용자 경험(UX) 개선 포인트

### 2.1 즉시 개선 가능 (Quick Wins)

#### A. 햅틱 피드백 강화

**현재 상태**: 피드백 없음
**개선안**:
```dart
// 슛 기록 성공 시
HapticFeedback.mediumImpact();

// 슛 실패 시
HapticFeedback.lightImpact();

// 파울 5개 (퇴장) 시
HapticFeedback.heavyImpact();
HapticFeedback.heavyImpact(); // 2연속
```

**효과**: 눈을 코트에 두면서도 기록 확인 가능

---

#### B. 제스처 단축키

**현재**: 모든 액션이 탭 기반
**개선안**:
```
- 선수 카드 왼쪽 스와이프 → 2점 성공 (빠른 기록)
- 선수 카드 오른쪽 스와이프 → 3점 성공
- 선수 카드 길게 누르기 → 전체 메뉴
- 코트 더블탭 → 팀 리바운드
```

**효과**: 자주 쓰는 액션 입력 시간 50% 단축

---

#### C. 다크모드 자동 전환

**현재**: 고정 다크 테마
**개선안**:
```dart
// 시간대별 자동 전환
final hour = DateTime.now().hour;
final brightness = (hour >= 18 || hour < 6)
    ? Brightness.dark
    : Brightness.light;

// 또는 시스템 설정 따르기
MediaQuery.platformBrightnessOf(context);
```

**효과**: 실내/실외, 낮/밤 상관없이 최적 가독성

---

### 2.2 단기 개선 (1-2주)

#### D. 선수 빠른 검색

**문제**: 선수가 많으면 스크롤 필요
**개선안**:
```
- 상단 검색바: 이름/등번호 검색
- 등번호 입력 → 즉시 포커스
- 예: "23" 입력 → 23번 선수 하이라이트
```

---

#### E. 스마트 어시스트 추천

**현재**: 성공 슛 후 모든 팀원 표시
**개선안**:
```
- 최근 30초 내 볼 터치한 선수 상위 표시
- 해당 경기 어시스트 많은 선수 상위 표시
- "어시스트 없음" 버튼 더 크게
```

---

#### F. 경기 시작 체크리스트

**문제**: 기록 시작 전 누락 가능
**개선안**:
```
경기 시작 체크리스트:
☑ 선발 5명 선택 (홈)
☑ 선발 5명 선택 (원정)
☑ 타이머 설정 확인 (10분 x 4쿼터)
☑ 네트워크 상태 확인
☐ 점프볼 승자 선택

[경기 시작] 버튼 활성화
```

## 결론

BDR Tournament Recorder는 **탄탄한 기술 기반**(오프라인 우선, 402개 테스트, 클린 아키텍처)을 갖추고 있습니다.

**차별화 전략**:
1. **속도**: "3초 안에 기록" 목표 유지
2. **신뢰성**: 오프라인에서도 데이터 유실 없음
3. **인사이트**: 단순 기록 → AI 기반 분석

**v2.0 비전**: "경기 기록"에서 "농구 데이터 플랫폼"으로 진화

다음 단계로 v1.0 릴리스 후 사용자 피드백을 수집하며 v1.1 Quick Wins를 병행 개발하는 것을 권장합니다.
