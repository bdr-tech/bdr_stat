# BDR Tournament Recorder - 요구사항 정의서 v2.0

> 농구 경기 점수 기록용 네이티브 앱 (Flutter)

---

## 1. 개요

### 1.1 목적
아마추어 농구 동호인 대회에서 실시간으로 경기 점수와 개인 기록을 효율적으로 기록하고, 
오프라인 환경에서도 안정적으로 데이터를 저장하며, 서버와 동기화하는 태블릿 전용 앱

### 1.2 핵심 가치
- **오프라인 우선**: 체육관의 불안정한 네트워크 환경에서도 데이터 손실 없이 기록
- **빠른 입력**: S Pen 에어 커맨드로 2액션 이내 기록 완료
- **정확한 데이터**: 개인 통계와 슛 위치 좌표까지 상세 기록

## 3. 사용 프로세스

### 3.1 전체 흐름

```
[대회 불러오기] → [경기 선택] → [로스터 확인] → [스타팅 선택] → [경기 기록] → [로컬 저장] → [서버 동기화]
```

### 3.2 상세 프로세스

#### Step 1: 대회 불러오기
1. 앱 실행 후 로그인 (기록원 계정)
2. "대회 불러오기" 버튼 클릭
3. 서버에서 **당일 일정이 있는 경기 목록** 조회
4. 경기 목록 로컬 캐싱

#### Step 2: 경기 선택
1. 경기 리스트에서 기록할 경기 선택
2. 경기 시간 검증
   - 현재 시간이 경기 예정 시간과 맞지 않으면 → 확인 얼럿 표시
   - "경기 시간이 아닙니다. 그래도 시작하시겠습니까?"

#### Step 3: 로스터 불러오기
1. 양 팀 등록 선수 목록 조회
2. **필수 조건**: 모든 선수는 MyBDR 서비스에 가입되어 있어야 함
3. 미가입 선수는 로스터에 표시되지 않음

#### Step 4: 스타팅 멤버 선택
1. 각 팀별 5명의 선발 선수 선택
2. 선택된 선수는 코트 위 포지션 원으로 표시
3. 벤치 선수는 사이드 패널에 대기

#### Step 5: 경기 기록
1. 실시간 점수 및 개인 기록 입력
2. 모든 기록은 **즉시 로컬 저장** (SQLite/Drift)
3. 쿼터 종료 시 자동 저장 체크포인트

#### Step 6: 경기 종료 및 동기화
1. 경기 종료 버튼 클릭
2. 최종 박스스코어 확인
3. "서버 전송" 버튼으로 MyBDR API에 데이터 동기화
4. 동기화 실패 시 로컬에 보관, 재시도 가능

## 4. UI 구성

### 4.1 전체 레이아웃 (가로 모드)

```
┌─────────────────────────────────────────────────────────────────┐
│ [대회명] [날짜] [경기: Team A vs Team B]           [네트워크 상태] │  ← 상단 정보바
├─────────────────────────────────────────────────────────────────┤
│   [경기시간 10:00]  [Q1]  [공격시간 24]   [⏸️][▶️][⏹️]          │  ← 타이머 컨트롤
├──────────┬─────────────────────────────────┬────────────────────┤
│          │                                 │                    │
│  TEAM A  │                                 │  TEAM B            │
│  로스터   │      🏀 농구 코트 (가로)         │  로스터             │
│          │                                 │                    │
│  2:      │         (슛 위치 기록)           │              :2    │
│          │                                 │                    │
│  6:      │      ● ●     ●     ● ●         │              :6    │
│  비율    │      (선수 포지션 표시)          │              비율   │
│          │                                 │                    │
├──────────┴─────────────────────────────────┴────────────────────┤
│ [팀A 파울: 2] [팀A 점수: 24]     [팀B 점수: 21] [팀B 파울: 3]     │  ← 스코어보드
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 영역별 상세

#### 상단 정보바
| 요소 | 설명 |
|------|------|
| 대회명 | 현재 진행 중인 대회 이름 |
| 날짜 | 경기 일자 |
| 경기 정보 | "Team A vs Team B" 형식 |
| 네트워크 상태 | 🟢 온라인 / 🔴 오프라인 인디케이터 |

#### 타이머 컨트롤 영역
| 요소 | 설명 | 인터랙션 |
|------|------|----------|
| 경기 시간 | 남은 시간 표시 (기본 10:00) | 탭: 시작/일시정지, 롱프레스: 설정 |
| 쿼터 | Q1~Q4 (또는 연장) | 탭: 다음 쿼터 |
| 공격 시간 | 샷클락 (기본 24초) | 탭: 리셋, 롱프레스: 설정 |
| 제어 버튼 | 일시정지/재개/종료 | - |

#### 시간 설정 (롱프레스 시 팝업)
- 경기 시간: 5분 / 8분 / 10분 / 12분 / 사용자 지정
- 공격 시간: 14초 / 24초 / 30초 / 없음
- 쿼터 수: 2쿼터(전후반) / 4쿼터

#### 양쪽 로스터 패널 (2:6:2 비율의 양쪽 2)
```
┌─────────────┐
│ #23 홍길동   │  ← 등번호 | 선수명
│ PG | 12pts  │  ← 포지션 | 현재 기록 요약
├─────────────┤
│ #7 김철수   │
│ SG | 8pts   │
├─────────────┤
│    ...      │
├─────────────┤
│ ── 벤치 ──  │  ← 구분선
│ #15 이영희  │
│ SF | 0pts   │
└─────────────┘
```

- **상단 5명**: 현재 코트에 있는 선수 (ON COURT)
- **하단**: 벤치 선수 (BENCH)
- **교체**: 드래그 앤 드롭 또는 두 선수 순차 탭

#### 중앙 코트 영역 (6)
- 농구 코트를 가로로 배치
- 선수 위치를 원(●)으로 표시 (작전판 스타일)
- 원 안에 등번호 표시
- **홈팀/어웨이팀 색상 구분** (유니폼 색상 반영)
- 슛 위치 터치 시 → 에어 커맨드 메뉴 호출

## 5. 점수 기록 시스템

### 5.1 에어 커맨드 메뉴

코트 영역 터치 또는 S Pen 버튼 클릭 시 원형 메뉴 표시:

```
        [2점 성공]
    [3점 실패]  [3점 성공]
  [리바운드]      [어시스트]
    [블락]    [스틸]
  [파울]        [턴오버]
        [2점 실패]
```

### 5.2 기록 항목

| 카테고리 | 기록 항목 | 상세 |
|----------|----------|------|
| **득점** | 1점 (자유투) | 성공/실패 |
| | 2점 (필드골) | 성공/실패, 슛 위치 XY 좌표 |
| | 3점 (장거리) | 성공/실패, 슛 위치 XY 좌표 |
| **리바운드** | 공격 리바운드 | - |
| | 수비 리바운드 | - |
| **어시스트** | 어시스트 | 득점과 연계 기록 |
| **수비** | 블락 | - |
| | 스틸 | - |
| **실책** | 턴오버 | - |
| | 파울 | 개인 파울 카운트 증가 |

### 5.3 기록 흐름 (2액션 원칙)

**기본 흐름:**
```
1. 코트에서 위치 터치 → 에어 커맨드 표시
2. 기록 항목 선택 → 선수 자동 선택 또는 선수 선택 팝업
```

**선수 자동 선택:**
- 코트 위 선수 아이콘 근처 터치 시 해당 선수 자동 선택
- 멀리 터치 시 선수 선택 팝업 표시

### 5.4 슛 차트 데이터

모든 슛 시도는 다음 정보를 저장:
```json
{
  "player_id": "12345",
  "shot_type": "3PT",
  "result": "made",
  "x_coord": 0.75,
  "y_coord": 0.42,
  "quarter": 2,
  "game_time": "05:32",
  "timestamp": "2025-01-22T14:32:15Z"
}
```

## 6. 경기 운영 기능

### 6.1 팀 파울 관리

| 기능 | 설명 |
|------|------|
| 팀 파울 카운트 | 쿼터별 팀 파울 누적 표시 |
| 보너스 알림 | 팀 파울 5개 도달 시 "보너스" 표시 |
| 쿼터 리셋 | 새 쿼터 시작 시 팀 파울 0으로 리셋 |

### 6.2 개인 파울 관리

| 기능 | 설명 |
|------|------|
| 파울 카운트 | 선수 카드에 파울 개수 표시 (●●●○○) |
| 파울 트러블 | 4개 파울 시 경고 표시 (노란색) |
| 파울 아웃 | 5개 파울 시 퇴장 알림 + 강제 교체 팝업 |

### 6.3 작전 타임아웃

| 기능 | 설명 |
|------|------|
| 타임아웃 버튼 | 팀별 타임아웃 요청 버튼 |
| 잔여 표시 | 팀별 남은 타임아웃 개수 표시 |
| 시간 일시정지 | 타임아웃 시 자동으로 경기 시계 정지 |

### 6.4 공격권 (Possession)

| 기능 | 설명 |
|------|------|
| 화살표 표시 | 현재 공격권 팀 방향 화살표 |
| 자동 전환 | 점프볼/헬드볼 후 번갈아 전환 |
| 수동 전환 | 탭하여 수동 변경 가능 |

### 6.5 쿼터/하프타임 관리

| 기능 | 설명 |
|------|------|
| 쿼터 종료 | 시간 0:00 도달 시 자동 알림 |
| 휴식 타이머 | 쿼터 간 휴식 시간 카운트다운 (선택) |
| 진영 교체 | 전반/후반 시 코트 방향 자동 전환 |
| 연장전 | 동점 시 연장전 추가 옵션 |

## 8. 데이터 저장 및 동기화

### 8.1 로컬 저장 전략

| 시점 | 저장 내용 |
|------|----------|
| 매 기록 즉시 | 개별 플레이 기록 |
| 쿼터 종료 시 | 체크포인트 (전체 상태 스냅샷) |
| 경기 종료 시 | 최종 박스스코어 + 모든 기록 |
| 앱 백그라운드 시 | 현재 상태 자동 저장 |

### 8.2 데이터 구조

```
LocalDB (SQLite/Drift)
├── games/           # 경기 기본 정보
├── game_events/     # 플레이 바이 플레이 기록
├── box_scores/      # 박스스코어 집계
├── shot_charts/     # 슛 위치 좌표
├── sync_queue/      # 동기화 대기열
└── edit_logs/       # 수정 이력
```

### 8.3 서버 동기화

| 기능 | 설명 |
|------|------|
| 수동 동기화 | "서버 전송" 버튼 클릭 시 |
| 자동 재시도 | 실패 시 로컬 큐에 보관, 네트워크 복구 시 재시도 |
| 충돌 해결 | 타임스탬프 기반 Last-Write-Wins |
| 동기화 상태 | 각 경기별 동기화 완료/대기/실패 표시 |

## 9. 박스스코어 출력

### 9.1 박스스코어 형식 (NBA.com 스타일)

```
┌─────────────────────────────────────────────────────────────────────────┐
│ TEAM A                                                                   │
├─────┬──────────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┬───────┤
│ #   │ PLAYER   │ MIN │ PTS │ REB │ AST │ STL │ BLK │ TO  │ PF  │ FG    │
├─────┼──────────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┼───────┤
│ 23  │ 홍길동   │ 32  │ 24  │ 5   │ 3   │ 2   │ 0   │ 1   │ 2   │ 8/15  │
│ 7   │ 김철수   │ 28  │ 18  │ 8   │ 2   │ 1   │ 1   │ 2   │ 3   │ 6/12  │
│ ... │ ...      │ ... │ ... │ ... │ ... │ ... │ ... │ ... │ ... │ ...   │
├─────┼──────────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┼───────┤
│     │ TOTALS   │ 200 │ 78  │ 35  │ 18  │ 7   │ 3   │ 10  │ 15  │ 28/60 │
└─────┴──────────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┴───────┘
```

### 9.2 상세 통계 (선택적 확장)

- FG%: 필드골 성공률
- 3P%: 3점슛 성공률
- FT%: 자유투 성공률
- OREB/DREB: 공격/수비 리바운드 구분
- +/-: 플러스마이너스

## 10. 추가 고려사항

### 10.1 사용자 권한

| 역할 | 권한 |
|------|------|
| 기록원 | 경기 기록, 수정, 로컬 저장 |
| 대회 관리자 | 기록원 권한 + 서버 동기화, 경기 생성/수정 |
| 슈퍼 관리자 | 모든 권한 + 기록 삭제, 데이터 초기화 |

### 10.2 에러 처리

| 상황 | 대응 |
|------|------|
| 네트워크 끊김 | 오프라인 모드 전환, 로컬 저장 계속 |
| 앱 크래시 | 자동 복구, 마지막 저장 시점부터 재개 |
| 동기화 실패 | 로컬 큐 보관, 재시도 버튼 제공 |
| 데이터 충돌 | 충돌 알림, 수동 해결 옵션 |

### 10.3 접근성

- 큰 터치 영역 (최소 44x44pt)
- 고대비 색상 옵션
- 왼손잡이 모드 (UI 좌우 반전)

### 10.4 경기 중단/재개

| 기능 | 설명 |
|------|------|
| 임시 저장 | 경기 중 "일시 저장" 가능 |
| 이어서 기록 | 저장된 경기 선택하여 재개 |
| 상태 복원 | 점수, 시간, 선수 배치 모두 복원 |

## 11. 기술 스택

### 11.1 선정 기술

| 영역 | 기술 | 선정 이유 |
|------|------|----------|
| 프레임워크 | Flutter | 크로스 플랫폼, 네이티브 성능 |
| 상태관리 | Riverpod | 복잡한 상태 관리에 적합 |
| 로컬 DB | Drift (SQLite) | 오프라인 우선, 트랜잭션 안전성 |
| HTTP | Dio | 인터셉터, 재시도 로직 |
| API | Rails (MyBDR) | 기존 서비스 연동 |

### 11.2 PWA 미선정 사유

| 요구사항 | PWA 한계 |
|----------|----------|
| S Pen 에어 커맨드 | 삼성 SDK 접근 불가 (네이티브 필수) |
| 안정적 오프라인 저장 | IndexedDB < SQLite 안정성 |
| iOS 백그라운드 Sync | Safari 제한적 지원 |
