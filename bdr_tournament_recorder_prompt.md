# BDR Tournament Recorder - 태블릿 전용 대회 기록 앱 개발 프롬프트

## 📋 프로젝트 개요

BDR(Basketball Daily Routine) 플랫폼의 대회 경기 기록을 위한 **태블릿 전용 앱**을 개발해주세요. 
이 앱은 체육관 등 네트워크 환경이 불안정한 곳에서 사용되므로 **오프라인 우선(Offline-First)** 아키텍처가 핵심입니다.

---

## 🛠 기술 스택

### 필수 기술
- **Framework**: Flutter (iOS/Android 크로스플랫폼)
- **상태관리**: Riverpod
- **로컬 DB**: Drift (SQLite ORM)
- **HTTP Client**: Dio
- **화면 방향**: 가로 모드(Landscape) 고정

### 백엔드 연동
- **Backend**: Ruby on Rails 7 API
- **Database**: PostgreSQL
- **동기화 방식**: 경기 종료 후 일괄 전송 (Batch Sync)

## 📱 핵심 요구사항

### 1. 오프라인 우선 아키텍처
```
[요구사항]
- 네트워크 연결 없이도 모든 기록 기능이 정상 동작해야 함
- 모든 데이터는 로컬 SQLite에 즉시 저장
- 경기 종료 후 네트워크 연결 시 서버에 일괄 동기화
- 동기화 충돌 시 로컬 데이터 우선 (Last Write Wins with timestamp)
- 동기화 상태 표시 (미동기화 건수, 마지막 동기화 시간)
```

### 2. 입력 장치 최적화
```
[요구사항]
- 태블릿 펜(스타일러스) 사용에 최적화된 UX
- 터치 타겟 최소 크기: 48x48dp (펜 정밀도 고려)
- 에어 커맨드 지원 (펜 근접 시 퀵메뉴 자동 표시)
- 제스처: 탭, 롱프레스, 스와이프 지원
- 실수 방지를 위한 확인 피드백 (햅틱/시각적)
```

### 2-1. 에어 커맨드 (Air Command) 상세
```
[목표]
- 기록 입력을 "원-투" 2동작으로 완료
- 펜 호버 → 바로 액션 탭 (탭해서 메뉴 여는 동작 생략)

[동작 원리]
1. 펜이 선수 카드 위 15-20mm 근접 감지
2. 해당 선수의 퀵 액션 메뉴가 자동 팝업
3. 메뉴에서 원하는 액션 바로 탭
4. 펜이 멀어지면 메뉴 자동 닫힘 (300ms 딜레이)

[퀵 액션 메뉴 레이아웃 - 방사형]
                 ┌─────┐
                 │ 3점  │
                 └──┬──┘
        ┌─────┐    │    ┌─────┐
        │ 2점  │────┼────│ FT  │
        └─────┘    │    └─────┘
             ┌─────┴─────┐
             │  #7 김가드 │  ← 선수 카드 (중심)
             └─────┬─────┘
        ┌─────┐    │    ┌─────┐
        │리바운│────┼────│ 파울 │
        └─────┘    │    └─────┘
                 ┌──┴──┐
                 │ 기타 │ → 스틸, 블락, 턴오버, 어시스트
                 └─────┘

[액션별 후속 동작]
- 2점/3점 탭 → 코트에서 위치 탭 → 성공/실패 선택
- FT 탭 → 자유투 개수 → 순차 입력
- 파울 탭 → 파울 종류 선택
- 리바운드 탭 → 공격/수비 선택
- 기타 탭 → 서브메뉴 (스틸, 블락, 턴오버, 어시스트)

[코트 위 에어 커맨드]
펜이 코트 위 호버 시:
- 현재 존(Zone) 번호 표시
- 3점 라인 안/밖 표시
- 예상 거리 표시 (예: "7.2m")

[Flutter 구현]
- Listener 위젯의 onPointerHover 이벤트 사용
- PointerHoverEvent.kind == PointerDeviceKind.stylus 체크
- position으로 호버 위치 감지
- Overlay 또는 Stack으로 메뉴 표시

[예시 코드 구조]
class PlayerCardWithAirCommand extends StatefulWidget {
  @override
  Widget build(BuildContext context) {
    return Listener(
      onPointerHover: (event) {
        if (event.kind == PointerDeviceKind.stylus) {
          // 펜 호버 감지 - 메뉴 표시
          _showQuickActionMenu(event.position);
        }
      },
      onPointerExit: (event) {
        // 펜 벗어남 - 300ms 후 메뉴 닫기
        _scheduleMenuClose();
      },
      child: PlayerCard(...),
    );
  }
}

[비-펜 입력 대응]
- 손가락 터치: 기존 방식 유지 (탭 → 메뉴)
- 마우스: 호버로 메뉴 표시 (테스트/개발용)
- 펜 없는 태블릿: 롱프레스로 메뉴 표시
```

### 2-2. 입력 속도 최적화 목표
```
[목표 시간]
- 단순 슛 기록 (위치 없이): 2초 이내
- 슛 + 위치 기록: 3초 이내  
- 슛 + 위치 + 어시스트: 4초 이내
- 자유투 2개: 3초 이내
- 선수 교체 (1:1): 3초 이내

[입력 동작 수]
┌─────────────────────────────────────────────────────────┐
│ 액션                    │ 에어커맨드  │ 기존(탭방식)   │
├─────────────────────────────────────────────────────────┤
│ 2점슛 성공 (위치 없이)   │ 2동작      │ 3동작         │
│ 3점슛 성공 + 위치        │ 3동작      │ 4동작         │
│ 3점슛 + 위치 + 어시스트  │ 4동작      │ 5동작         │
│ 자유투 2개 (1성공 1실패) │ 4동작      │ 5동작         │
│ 리바운드               │ 2동작      │ 3동작         │
│ 파울                   │ 2동작      │ 3동작         │
└─────────────────────────────────────────────────────────┘

* 에어커맨드로 평균 1동작 절약 → 경기당 수백 번 = 상당한 시간 절약
```

```dart
int calculatePoints(PlayerStats stats) {

         (stats.threePointersMade * 3) +

// 슛 종류별 필드 업데이트
void recordShot(ShotType type, bool made) {
  switch (type) {
    case ShotType.twoPoint:
      stats.fieldGoalsAttempted++;
      stats.twoPointersAttempted++;
      if (made) {
        stats.fieldGoalsMade++;
        stats.twoPointersMade++;
        stats.points += 2;
      }
      break;
    case ShotType.threePoint:
      stats.fieldGoalsAttempted++;
      stats.threePointersAttempted++;
      if (made) {
        stats.fieldGoalsMade++;
        stats.threePointersMade++;
        stats.points += 3;
      }
      break;
    case ShotType.freeThrow:
      stats.freeThrowsAttempted++;
      if (made) {
        stats.freeThrowsMade++;
        stats.points += 1;
      }
      break;
  }
}
```

```dart
class TeamFoulManager {
  static const int BONUS_THRESHOLD = 5;  // 5파울부터 보너스
  
  Map<int, int> quarterFouls = {};  // 쿼터별 팀 파울
  
  void addFoul(int quarter, int teamId) {
    quarterFouls[quarter] = (quarterFouls[quarter] ?? 0) + 1;
  }
  
  bool isInBonus(int quarter, int teamId) {
    return (quarterFouls[quarter] ?? 0) >= BONUS_THRESHOLD;
  }
  
  void resetForNewQuarter(int newQuarter) {
    quarterFouls[newQuarter] = 0;
  }
  
  // 일반 파울 시 보너스 체크
  void onNonShootingFoul(int quarter, int teamId, int fouledPlayerId) {
    addFoul(quarter, teamId);
    
    if (isInBonus(quarter, teamId)) {
      // 자유투 2개 시퀀스 시작
      startFreeThrowSequence(fouledPlayerId, 2);
    }
  }
}
```

## 👤 User 누적 스탯 연동

### 개요

### 신규 테이블: user_career_stats (통산 기록)
```ruby
# 모든 대회 통산 기록 집계
class UserCareerStat < ApplicationRecord
  belongs_to :user
  
  # match_player_stats 생성 시 트리거
  def self.update_from_match(user_id, match_stat)
    career = find_or_create_by(user_id: user_id)
    
    career.total_games += 1
    career.total_games_started += 1 if match_stat.is_starter
    career.total_minutes += match_stat.minutes_played
    
    # 득점
    career.total_points += match_stat.points
    career.total_fg_made += match_stat.field_goals_made
    career.total_fg_attempted += match_stat.field_goals_attempted
    career.total_2pt_made += match_stat.two_pointers_made
    career.total_2pt_attempted += match_stat.two_pointers_attempted
    career.total_3pt_made += match_stat.three_pointers_made
    career.total_3pt_attempted += match_stat.three_pointers_attempted
    career.total_ft_made += match_stat.free_throws_made
    career.total_ft_attempted += match_stat.free_throws_attempted
    
    # 리바운드
    career.total_offensive_rebounds += match_stat.offensive_rebounds
    career.total_defensive_rebounds += match_stat.defensive_rebounds
    career.total_rebounds += match_stat.total_rebounds
    
    # 기타
    career.total_assists += match_stat.assists
    career.total_steals += match_stat.steals
    career.total_blocks += match_stat.blocks
    career.total_turnovers += match_stat.turnovers
    career.total_fouls += match_stat.personal_fouls
    
    # 평균 재계산
    career.recalculate_averages!
    
    # 커리어 하이 체크
    career.check_career_highs!(match_stat)
    
    career.last_calculated_at = Time.current
    career.save!
  end
  
  def recalculate_averages!
    return if total_games.zero?
    
    self.ppg = (total_points.to_f / total_games).round(1)
    self.rpg = (total_rebounds.to_f / total_games).round(1)
    self.apg = (total_assists.to_f / total_games).round(1)
    self.fg_percentage = total_fg_attempted > 0 ? 
      (total_fg_made.to_f / total_fg_attempted * 100).round(1) : 0.0
    self.three_pt_percentage = total_3pt_attempted > 0 ?
      (total_3pt_made.to_f / total_3pt_attempted * 100).round(1) : 0.0
    self.ft_percentage = total_ft_attempted > 0 ?
      (total_ft_made.to_f / total_ft_attempted * 100).round(1) : 0.0
  end
  
  def check_career_highs!(match_stat)
    today = Date.current
    
    if match_stat.points > career_high_points
      self.career_high_points = match_stat.points
      self.career_high_points_date = today
    end
    
    if match_stat.total_rebounds > career_high_rebounds
      self.career_high_rebounds = match_stat.total_rebounds
      self.career_high_rebounds_date = today
    end
    
    if match_stat.assists > career_high_assists
      self.career_high_assists = match_stat.assists
      self.career_high_assists_date = today
    end
  end
end
```

### 동기화 실패 처리
```dart
class SyncManager {
  // 재시도 로직
  Future<SyncResult> syncMatch(LocalMatch match) async {
    const maxRetries = 3;
    var retryCount = 0;
    
    while (retryCount < maxRetries) {
      try {
        final result = await _apiClient.batchSync(
          match: match.toServerPayload(),
          playerStats: await _getPlayerStats(match.id),
          playByPlays: await _getPlayByPlays(match.id),
        );
        
        // 성공 시 로컬 상태 업데이트
        await _markAsSynced(match.id, result.serverMatchId);
        return SyncResult.success(result);
        
      } on NetworkException catch (e) {
        retryCount++;
        if (retryCount >= maxRetries) {
          // 실패 기록
          await _markSyncFailed(match.id, e.message);
          return SyncResult.failed(e.message);
        }
        await Future.delayed(Duration(seconds: pow(2, retryCount).toInt()));
      }
    }
  }
  
  // 미동기화 경기 목록 조회
  Future<List<LocalMatch>> getUnsyncedMatches() {
    return _database.localMatches
      .where((m) => m.isSynced.equals(false) & m.status.equals('completed'))
      .get();
  }
}
```

```dart
class ShotChartPainter extends CustomPainter {
  final List<ShotData> shots;
  final ShotChartMode mode; // scatter, heatmap, zone
  final ShotChartFilters filters;
  
  @override
  void paint(Canvas canvas, Size size) {
    _drawCourt(canvas, size);      // 코트 라인
    _drawThreePointLine(canvas, size);
    _drawPaintArea(canvas, size);
    _drawRestrictedArea(canvas, size);
    
    switch (mode) {
      case ShotChartMode.scatter:
        _drawShotMarkers(canvas, size);
        break;
      case ShotChartMode.heatmap:
        _drawHeatMapZones(canvas, size);
        break;
      case ShotChartMode.zone:
        _drawZoneOverlay(canvas, size);
        break;
    }
  }
  
  // 좌표 변환: 실제 코트 (94ft x 50ft) → 화면 픽셀
  Offset _courtToScreen(double courtX, double courtY, Size size) {
    // courtX, courtY는 0-100 정규화된 값
    return Offset(
      courtX / 100 * size.width,
      courtY / 100 * size.height,
    );
  }
}
```

## 🚨 주의사항

### 기존 스키마 관련
1. **tournaments.id는 UUID**: bigint가 아닌 UUID 타입. 로컬 DB에서 TEXT로 처리
2. **tournament_matches.uuid**: 경기 식별에 uuid 필드 사용 (id와 별개)
3. **tournament_team_players.id**: 선수 식별의 핵심 키. user_id가 아닌 이 값으로 통계 연결
4. **match_player_stats**: 이미 상세 컬럼 존재. 동일 구조로 로컬 DB 설계
5. **quarter_scores**: JSONB로 쿼터별 점수 저장. 구조 정확히 맞출 것

### 퍼포먼스
6. **PlayByPlay 테이블**: 경기당 수백 건 발생. 인덱싱 필수 (match_id, quarter, game_clock)
7. **로컬 DB 최적화**: Drift의 인덱스 활용, 불필요한 조인 최소화
8. **대량 INSERT**: batch insert 사용 (동기화 시)

### 데이터 무결성
9. **박스스코어 수동 수정**: PlayByPlay와 불일치 가능 → isManuallyEdited 플래그로 표시
10. **동기화 중복 방지**: local_id(UUID)로 멱등성 보장
11. **트랜잭션**: 경기 데이터 동기화는 반드시 트랜잭션으로 처리

### UX/디바이스
12. **배터리**: 타이머 상시 실행으로 배터리 소모 주의. 화면 밝기 조절 옵션 제공
13. **실수 방지**: 경기 종료, 데이터 삭제 등 중요 액션은 반드시 확인 다이얼로그
14. **테스트**: 오프라인 상태에서 충분한 테스트 필수
15. **태블릿 가로모드**: 회전 잠금 필수, 펜 입력 최적화

### API 인증
16. **api_token 보안**: 토큰은 로컬에 안전하게 저장 (flutter_secure_storage)
17. **토큰 만료 처리**: 401 응답 시 재인증 플로우

### UX 플로우 관련
18. **액션 순서**: 슛 → 위치 → 성공/실패 → 어시스트(성공시)/리바운드(실패시) 순서 고정
19. **연동 자동화**: 스틸→턴오버, 블락→슛실패, 오펜시브파울→턴오버 자동 연결
20. **보너스 상태**: 팀파울 5개 이상 시 자동 전환, 일반 파울도 자유투 2개
21. **쿼터 리셋**: 팀파울은 쿼터마다 리셋, 개인파울은 누적
22. **자유투 시퀀스**: 2개/3개 자유투는 연속 입력, 마지막 실패 시 리바운드 연결
23. **교체 검증**: OUT 수 = IN 수 일치해야 확정 가능
24. **5파울 아웃**: 자동 감지 후 강제 교체 모달 표시

### 동기화 결과 표시 (핵심!)
25. **개인 기록 업데이트 확인**: 동기화 완료 후 반드시 선수별 통산 기록 변화 표시
26. **커리어 하이 알림**: 개인 신기록 달성 시 특별 알림
27. **PPG 변화**: 통산 평균 변화량 표시 (+0.1 등)
